🧠 RAG-память Кеши

Кеша запоминает ВСЮ историю диалогов и находит нужное по смыслу — даже после сжатия контекста. Вместо «забыл после компакта» — «нашёл в 247 сообщений назад, вот цитата».

😤 Проблема: контекст теряется

Сейчас Кеша помнит только текущую сессию. Когда контекст заполняется на 95% — происходит compact: всё сжимается в краткое саммари. Детали исчезают навсегда.

Сейчас (без RAG)
  • Compact → детали теряются
  • «Что я ел в среду?» → «хз, не записал»
  • «Когда мы ссорились?» → «в саммари этого нет»
  • Каждая сессия с нуля
  • Логи есть, но только превью
С RAG-памятью
  • Compact не страшен — память в БД
  • «Что я ел?» → цитата с датой
  • «Когда ссорились?» → точные сообщения
  • Помнит ВСЮ историю навсегда
  • Поиск по смыслу, не по словам

Что такое Embedding?

Это «числовой отпечаток смысла». Модель читает текст и превращает его в массив из 384 чисел. Похожие по смыслу тексты получают похожие числа — даже если слова разные.

"поссорились из-за посуды" → [0.12, -0.45, 0.78, ..., 0.33]
"конфликт на кухне" → [0.11, -0.43, 0.76, ..., 0.31] ← похожие!
"купил новый ноутбук" → [-0.67, 0.22, -0.11, ..., 0.89] ← далеко

🏗️ Выбранный стек

Три компонента — минимум зависимостей, всё локально, $0:

🗄️
sqlite-vec
Расширение SQLite для векторов. Тот же файл что messages.db — ноль новых серверов. pip install и готово.
🧠
mE5-small
Модель для embedding. 120MB RAM, понимает русский, работает на CPU за 15-40мс. Через FastEmbed (без PyTorch).
🔧
MCP tool
search_memory — Кеша сам решает когда искать. Не тратит контекст на каждое «привет».
Почему именно эти? (подробнее)

sqlite-vec вместо ChromaDB — ChromaDB тащит PyTorch (~2GB), держит всё в RAM, отдельный процесс. sqlite-vec — расширение нашего SQLite, один файл, ноль RAM overhead. На 100K сообщений brute-force скан за 10-40мс.

mE5-small вместо mE5-large — large жрёт 2.2GB RAM и 100-250мс на embedding. Small — 120MB и 15-40мс. На коротких чат-сообщениях разница в качестве минимальна.

MCP tool вместо auto-inject — auto-inject ищет и вставляет контекст на КАЖДОМ сообщении. На «ок», «да», «привет» — бессмысленный расход CPU и контекста. Tool = Кеша сам решает.

💡 Вывод

ANN-индекс (ChromaDB/LanceDB/zvec) нужен от миллионов векторов. У нас 100K максимум — brute-force sqlite-vec справляется за 10-40мс. Зачем тащить отдельный сервер ради того что SQLite делает из коробки?
